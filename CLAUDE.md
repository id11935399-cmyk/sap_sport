# SPORT_FAST_SPLAV — сайт сплавов на байдарках

Статический сайт (HTML/CSS/JS, без сборки, без бэкенда). Открывается напрямую через `index.html` в браузере — сервер не нужен.

## Структура

```
index.html, about.html, booking.html, contacts.html, corporate.html,
faq.html, prices.html, reviews.html, safety.html   — корневые страницы
routes/                                             — 9 страниц маршрутов + index.html (каталог)
legal/                                              — оферта, политика конфиденциальности
css/style.css                                       — вся дизайн-система, один файл
js/main.js                                          — бургер-меню, FAQ-аккордеон, формы, scroll-reveal,
                                                       параллакс hero, магнитные кнопки, prefill брони
img/, video/                                        — медиа, см. ниже
```

## Дизайн-система (важно!)

Сайт прошёл через два дизайн-этапа. **Текущая версия — v2**, ключевые принципы:
- **Никаких неоновых pill-кнопок со свечением** — это было в v1 и читалось как "AI-лендинг". Кнопки теперь прямоугольные (`--radius-sm`), со стрелкой `→`, без box-shadow-неона.
- **Никаких плейсхолдер-заглушек `.ph`, где можно поставить реальное фото.** Реальные фото — через классы `.media` (фото в блоке текст+картинка) и `.tile` (плитка маршрута — фото на всю площадь + градиент + текст поверх, без белой карточки-коробки).
- **Кинематографичный hero** — класс `.hero-cinematic` (не `.hero`/`.hero-sm`, это старые v1-классы, ещё используются кое-где по мелочи): полноэкранное фото/видео, градиент-скрим, `data-reveal` анимации, `<div class="scroll-cue">`. Хедер на таких страницах — `<header class="site-header transparent">` (прозрачный поверх фото, затемняется при скролле через JS).
- **`data-reveal`** на элементах — плавное появление при скролле (IntersectionObserver в main.js). `data-reveal="scale"` — с лёгким масштабированием.
- **Типографика**: `Bebas Neue` — заголовки (uppercase по умолчанию), `Manrope` — текст, `Fraunces` (курсив) — лид-абзацы и подзаголовки для "премиального" акцента (см. `.hero-title-sub`, `.lead`, `.pull-quote`).
- Индекс-лейблы вместо одинаковых eyebrow-бейджей на каждой секции: `<div class="index-label"><b>01</b> Название</div>`.

**Если чинишь/добавляешь страницу — используй `.hero-cinematic` + `.tile`/`.media`, а не старые `.hero`/`.card`/`.ph`.** Старые классы остаются в CSS ради обратной совместимости с ещё не до-переделанными страницами (serebryany-bor-loop.html, faq.html, prices.html, booking.html, reviews.html, safety.html, about.html hero — их можно доделать в том же стиле при следующей возможности).

## Медиа: img/ и video/

Структура папок:
```
img/  video/
  hero/                    — материалы для hero главной
  gallery/                 — общие атмосферные кадры (не привязаны к одному маршруту)
  about/                   — фото основателя (Валерий)
  safety/                  — жилеты, инструктаж
  corporate/               — корпоративный тимбилдинг
  routes/
    nerskaya/  serebryany-bor/  zhivopisny-most/
    arkhangelskoye/  serebryany-bor-loop/  night-photo-tour/
```

**Важно про рабочий процесс с фото:** пользователь физически перетаскивает файлы в эти папки сам (у Claude нет инструмента, который сохраняет вложенные в чат картинки на диск — их можно только посмотреть через Read, не скопировать). Пользователь скидывает фото под случайными именами вида `photo_2026-07-08_18-50-17.jpg` — их нужно:
1. Прочитать через Read, оценить содержание;
2. Скопировать (`Copy-Item`, не переименовать/удалить оригинал) под осмысленным именем в нужный слот, который уже вписан в HTML (`hero.jpg`, `secondary.jpg`, `night-hero.jpg`, `toast.jpg` и т.д. — имена см. прямо в `src=` атрибутах);
3. Прописать `<img src="...">` в HTML вместо `.ph`-плейсхолдера.

## Видео (ffmpeg)

Пользователь присылает большие RAW-видео с телефона (4K, 60-170 МБ). ffmpeg установлен через winget (`Gyan.FFmpeg`), путь:
```
C:\Users\User\AppData\Local\Microsoft\WinGet\Packages\Gyan.FFmpeg_Microsoft.Winget.Source_8wekyb3d8bbwe\ffmpeg-8.1.2-full_build\bin\ffmpeg.exe
```
Обработка для hero-видео: обрезать до ~15 сек, даунскейл до 1920×1080, убрать звук, `libx264 -crf 24 -preset slow -movflags +faststart`. Результат ~2 МБ. Постер-кадр — `ffmpeg -vframes 1` из готового ролика.
